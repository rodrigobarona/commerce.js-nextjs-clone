Prood packages are organized by responsibility. Applications compose them — no package depends on an app.

## Package dependency graph

```
@prood/types          (interfaces only — no dependencies)
    │
    ├── @prood/platform      (Neon Postgres engine)
    │       └── implements CommerceAdapter + AdminAPI
    │
    ├── @prood/commerce      (server data layer)
    │       └── wraps platform + payment/storage providers
    │
    ├── @prood/checkout      (state machine)
    │       └── uses PaymentProvider from types
    │
    ├── @prood/checkout-host (Redis session host)
    │       └── uses checkout + commerce
    │
    ├── @prood/payment-*     (gateway providers)
    ├── @prood/storage-*     (file upload providers)
    │
    ├── @prood/api-client    (typed HTTP client)
    └── @prood/ui            (React components)
```

## Which package do I need?

| Task | Package(s) |
| --- | --- |
| Build a storefront page | `@prood/api-client`, `@prood/ui`, `@prood/types` |
| Add a payment provider | `@prood/payment-*`, register in `@prood/commerce` |
| Custom commerce adapter | `@prood/types` (implement `CommerceAdapter`) |
| Direct database access | `@prood/platform` (prefer API for apps) |
| Hosted checkout session | `@prood/checkout-host` |
| Upload product images | `@prood/storage-*` via `@prood/commerce` |
