Prood is built for anyone who wants to **sell products online** without assembling a fragmented stack. This page summarizes what works today and what is still on the roadmap.

## What you get on day one

| Capability | Status |
| --- | --- |
| Register and create a store (organization) | Live |
| Instant subdomain `{your-slug}.prood.app` | Live |
| Product catalog with variants, images, inventory | Live |
| Customer storefront (browse, cart, checkout) | Live |
| Orders and customers in the dashboard | Live |
| Stripe, Easypay, or Ifthenpay per store | Live |
| Custom domain + SSL (production Vercel setup) | Live |
| Team invites (owner, admin, member) | Live |
| API keys for integrations | Live |

## What is coming soon

| Capability | Status |
| --- | --- |

## Typical first hour

1. [Register](https://admin.prood.app/register) on the dashboard (use your deployment URL in dev).
2. Add a few [products](/docs/apps/dashboard/products).
3. Connect [payments](/docs/apps/dashboard/integrations).
4. Share your subdomain URL with test buyers.
5. Complete a test [checkout](/docs/apps/storefront/cart-checkout) and confirm the order in the dashboard.

For the full checklist, see [Merchant onboarding](/docs/guides/merchant-onboarding).

## When to upgrade (planned plans)

Marketing pricing on [prood.app/pricing](https://prood.app/pricing) describes intended tiers:

- **Free** — subdomain, limited products and orders, solo operator.
- **Grow** — custom domain, Agent Auth, more team seats.
- **Scale** — higher volume and priority support.
- **Agency** — many isolated client stores.

Subscription billing in the dashboard is not live yet—you can run on Free while limits are being rolled out.
